# 使用文档

## 1. 登录方式

- 总主持/小组主持：账号密码登录（`/login`）
- 普通成员：成员直入链接（`/member-entry?token=...`）

## 2. 页面清单

1. 登录页（`/login`）
2. 会议门户首页（`/`）
3. 主会场页（`/meeting/main`）
4. 小组会议页（`/meeting/group/:groupId`）
5. 总主持控制台（`/console/master`）
6. 小组主持控制台（`/console/group`）
7. 配置编辑页（`/config/editor`）
8. 配置预览页（`/config/preview`）
9. 配置历史页（`/config/history`）
10. 系统日志页（`/logs`）
11. 矩阵巡检墙页（`/wall`）

## 3. 配置管理

- 左侧 YAML 编辑，右侧说明区
- 支持“校验配置”、“发布保存”、“载入当前配置”
- 每次保存会自动备份旧版本，默认保留最近 5 个
- 可在“配置历史页”执行一键回滚

## 4. 权限模型

- 总主持：全局分组/调度/录制/通知/配置编辑
- 小组主持：仅本组成员管理与本组控制
- 普通成员：仅入会与举手（受主持控制）

## 5. 控制动作执行机制

前端执行任何高权限动作前，先调用后端授权接口：

- `POST /api/control/authorize`

后端通过角色与目标房间做二次校验，授权通过后前端再执行 Jitsi 命令。
