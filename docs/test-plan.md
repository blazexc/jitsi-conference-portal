# 测试清单

## 配置测试

1. YAML 语法错误 -> 返回 errors
2. 缺失字段 -> 返回 errors
3. 用户角色非法 -> 返回 errors
4. breakout room ID 重复 -> 返回 errors
5. 用户所属组不存在 -> 返回 errors
6. 录制房间不存在 -> 返回 errors
7. 回滚后版本号递增且配置恢复

## 权限测试

1. 普通成员调用 `/api/control/authorize` 的高权限动作应被拒绝
2. 小组主持操作其他组房间应被拒绝
3. 总主持执行全局动作应通过
4. 非总主持访问配置接口应 403

## 会议联调测试

1. Jitsi iframe 正常加载
2. 主会场/小组会场可进入
3. 录制动作可发起授权请求并记录日志
4. 通知动作可发起授权请求并记录日志

## 矩阵墙测试

1. 2-6 画面展示
2. 分组切换正常
3. 轮播定时触发
4. 长时运行 2 小时无明显内存暴涨

## 部署验收

1. `GET /api/healthz` 返回 `ok=true`
2. `18080` 可访问门户
3. 反代后可访问 Jitsi 与门户
4. 文档步骤可复现
